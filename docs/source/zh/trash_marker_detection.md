# 废片标记识别功能使用说明

## 功能概述

废片标记识别功能可以自动识别口播视频中的废片标记词（如"这条不行"、"重来"、"不对"等），并自动删除对应的视频片段。这对于口播视频的后期剪辑非常有用，可以大大提高剪辑效率。

## 功能特点

1. **智能识别** - 基于LLM的语义理解，准确识别废片标记词
2. **自动删除** - 检测到废片标记后，自动删除对应的视频片段
3. **时间戳校准** - 删除废片后，自动校准后续片段的时间戳
4. **可配置** - 支持自定义废片标记词列表
5. **上下文感知** - 结合前后句子判断，避免误删

## 使用方法

### 1. 启用废片检测

在调用 `speech_rough_cut` 节点时，设置 `enable_trash_detection` 参数为 `true`：

```json
{
  "mode": "auto",
  "enable_trash_detection": true,
  "gap_threshold": 400,
  "user_request": ""
}
```

### 2. 自定义废片标记词

使用 `trash_markers` 参数指定要检测的废片标记词列表：

```json
{
  "mode": "auto",
  "enable_trash_detection": true,
  "trash_markers": ["这条不行", "重来", "不对", "算了", "不要这个", "重新开始", "再录一次", "从头来"],
  "gap_threshold": 400,
  "user_request": ""
}
```

### 3. 默认废片标记词

如果不指定 `trash_markers`，系统会使用以下默认标记词：

- 否定类："这条不行"、"不行"、"不对"、"错了"、"重来"、"算了"、"不要这个"
- 确认类（保留）："这条可以"、"这个好"、"就这样"、"OK"、"对"
- 重新开始类："重新开始"、"再录一次"、"从头来"

## 工作原理

### 处理流程

1. **ASR识别** - 首先使用 `LocalASRNode` 将语音转换为文字，获取精确的时间戳
2. **废片检测** - 对每个ASR句子，使用LLM检测是否包含废片标记词
3. **语义判断** - 结合上下文判断标记词的真实意图，避免误删
4. **片段删除** - 如果确认为废片，则跳过该片段，不包含在最终视频中
5. **时间戳校准** - 删除废片后，自动校准后续片段的时间戳
6. **视频裁剪** - 使用FFmpeg根据保留的片段时间戳裁剪视频

### 废片检测逻辑

系统会根据以下规则判断是否删除片段：

1. **精确匹配** - 句子中包含废片标记词
2. **语义判断** - 标记词用于否定当前或之前的内容
3. **上下文连贯** - 删除后前后句子仍然连贯
4. **避免误删** - 如果标记词出现在正常对话中，会保留

## 使用示例

### 示例1：基本使用

```python
# 在对话中启用废片检测
user_input = "帮我剪辑这个口播视频，自动删除包含'这条不行'的片段"

# 系统会自动调用 speech_rough_cut 节点
# 并设置 enable_trash_detection=true
```

### 示例2：自定义标记词

```python
user_input = "剪辑视频，删除所有包含'重来'、'不对'、'算了'的片段"

# 系统会使用用户指定的标记词列表
```

### 示例3：组合使用

```python
user_input = "剪辑这个口播视频，删除废片片段，同时去除语气词和重复内容"

# 系统会同时执行：
# 1. 废片标记检测
# 2. 语气词删除
# 3. 重复内容删除
```

## 注意事项

1. **上下文重要** - 系统会结合前后句子判断，确保不会误删正常内容
2. **标记词选择** - 选择合适的废片标记词很重要，避免过于宽泛导致误删
3. **时间戳精度** - ASR的时间戳精度会影响废片删除的准确性
4. **语音质量** - 清晰的语音可以提高ASR识别的准确性

## 技术实现

   - `src/open_storyline/nodes/node_schema.py` - 节点输入参数定义

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enable_trash_detection` | boolean | false | 是否启用废片检测 |
| `trash_markers` | List[str] | ["这条不行", "重来", ...] | 废片标记词列表 |
| `gap_threshold` | int | 400 | 分组间隔阈值（毫秒） |
| `user_request` | string | "" | 用户需求描述 |

## 常见问题

### Q: 为什么有些废片没有被删除？

A: 可能的原因：
1. 标记词不在检测列表中
2. 语音识别不准确，标记词没有被识别出来
3. 上下文判断认为该标记词不是废片标记

### Q: 为什么正常内容被删除了？

A: 可能的原因：
1. 标记词列表过于宽泛
2. 上下文判断出现误判
3. 建议：调整标记词列表，使其更加精确

### Q: 如何提高检测准确率？

A: 建议：
1. 使用清晰的语音录制
2. 选择合适的废片标记词
3. 在录制时使用明确的废片标记词
